# Frontend

Public-facing Next.js app for the Quant textbook site.

## Current architecture

- Routing: App Router TSX pages under `frontend/app/`.
- Chapter routes: `/chapters/{partSlug}/{chapterSlug}`.
- Shared docs/chapter shell: `frontend/components/layout/DocsShell.tsx`.
- Chapter renderer: `frontend/components/layout/ChapterPageLayout.tsx`.
- Glossary side panel: `frontend/components/glossary/`.

## Local development

From repo root:

```powershell
npm install
npm run sync:qmd
npm run dev
```

Open: `http://localhost:3000`

## Content workflow (QMD-first)

Write chapter content in `notebooks/chapters/**/*.qmd`, then sync into frontend TSX.

1. Edit chapter QMD content.
2. Optional frontmatter summary per chapter:

```yaml
---
title: "Chapter 1. Financial markets and instruments"
summary: "High-level overview of markets, stocks, ETFs, indexes, and benchmarks."
format:
  html: default
  pdf: default
---
```

3. Run:

```powershell
npm run sync:qmd
```

4. Reload dev server page.

### QMD parsing model

- `##` creates a chapter section.
- `###` creates a subsection (`h3`).
- Paragraphs, ordered lists, unordered lists, and fenced code blocks are preserved.
- Chapter summary is read from QMD frontmatter `summary:` and synced to `frontend/lib/chapterMetadata.ts`.

## Glossary system

### Source of truth

- Author glossary cards in:
  - `notebooks/chapters/part-10-glossary/chapter-47-glossary.qmd`
- Generated runtime data output:
  - `frontend/lib/generatedGlossary.ts`
- Runtime lookup helpers:
  - `frontend/lib/glossary.ts`

Do not manually edit `frontend/lib/generatedGlossary.ts`; it is generated by sync.

### Glossary card format in QMD

```md
**Volatility**
Aliases: Volatitly
Definition: ...
Intuition: ...
Why it matters: ...
Related terms: Risk, Variance, Standard Deviation
Chapter references: Chapter 3 (Risk, volatility, and drawdowns)
```

### Inline glossary tokens in chapter prose

Use these in QMD chapter text:

- `[[term]]`
- `[[term|display text]]`

Examples:

- `[[volatility]]`
- `[[market-capitalization|market cap]]`

Behavior:

- Resolved token: rendered as clickable glossary term.
- Unresolved token: rendered as plain text (safe fallback), with a dev console warning.
- Term matching is case-insensitive and alias-aware.

### Side panel behavior

- Opens on click/tap of a glossary term (no page navigation).
- Right-side drawer with slide-in/slide-out animation.
- Overlay fades in/out behind the drawer.
- Close methods:
  - Close button
  - `Escape`
  - Overlay click
  - Full glossary link (closes first, then navigates)
- Related term clicks update panel content in-place.
- State is local UI state (no URL query/hash sync for open/close state in v1).

Full glossary deep links use:

- `/chapters/part-10-glossary/glossary#term-<id>`

## Backend connection (optional for market widgets)

Set backend API base URL:

```powershell
$env:NEXT_PUBLIC_BACKEND_BASE_URL="http://127.0.0.1:8000"
```

Default fallback (if unset): `http://127.0.0.1:8000`

## Troubleshooting

- `Unable to acquire lock ... frontend/.next/dev/lock`
  - Another `next dev` is already running. Stop it, then run `npm run dev` again.
- Glossary token is not clickable
  - Ensure the term exists in glossary QMD, then run `npm run sync:qmd`.
- Chapter summary did not update
  - Add `summary:` to QMD frontmatter and rerun `npm run sync:qmd`.
